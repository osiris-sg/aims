AIMS Field App — How It Works

A technician's visit, from tapping the tag to a signed report.
The idea in one paragraph. A technician opens the app on their phone, taps it against the NFC sticker on a piece of equipment, and instantly sees that item's details. From there they can either confirm a recent delivery or write up a maintenance visit — adding photos, getting the customer to sign, and sending the report back to AIMS. The very first time a tag is tapped, the app asks the technician to type the item's SKU once, so it knows what the sticker belongs to from then on.

  participant T as 👷 Field Tech

  Note over T,B: Arriving on site

  T->>A: Opens AIMS and signs in
  A->>B: Checks who this user is
  B-->>A: "Field technician — show scan screen only"
  A-->>T: Shows the "Tap an asset tag" screen

  Note over T,B: Tapping the tag

  T->>A: Holds phone against the NFC sticker
  A->>A: Reads the tag's unique number
  A->>B: "What item is this tag for?"

  alt First time this tag has ever been tapped
    B-->>A: "Unknown tag"
    A-->>T: Shows "New tag — what item is this?"
    T->>A: Types the SKU printed on the item
    A->>B: "Find this item"
    B-->>A: Returns the item details
    A-->>T: Shows item card with a confirm button
    T->>A: Taps "Link this tag to this item"
    A->>B: Records that tag ↔ item link
    B-->>A: Saved
    Note over A: Future taps of this tag will skip
this whole step and go straight in.
  else Tag is already linked to an item
    B-->>A: Returns the item details
  end

  A->>B: Loads item info + recent delivery + past service reports
  B-->>A: Returns everything in one go
  A-->>T: Shows the item and two big buttons

  Note over T,B: Choosing what to do

  alt Doing a maintenance visit
    T->>A: Taps "Maintenance service report"
    A-->>T: Opens the report form
    T->>A: Types what was serviced / fixed
    loop For each photo of the work
      T->>A: Snaps a photo with the camera
      A->>B: Uploads the photo
      B-->>A: Stored
    end
    T->>A: Taps "Continue to signature"
    A->>B: Saves a draft report
    B-->>A: Draft saved
  else Acknowledging a delivery
    T->>A: Taps "Acknowledge latest Delivery Order"
    A-->>T: Shows the delivery info + form
    T->>A: Adds optional notes
    loop For each proof-of-delivery photo
      T->>A: Snaps a photo
      A->>B: Uploads the photo
      B-->>A: Stored
    end
    T->>A: Taps "Continue to signature"
    A->>B: Saves a draft (marked as delivery acknowledgement)
    B-->>A: Draft saved
  end

  Note over T,B: Getting the signature

  A-->>T: Shows the signature pad
  T->>A: Hands phone to customer / recipient
  T->>A: Customer draws their signature on screen
  T->>A: (Optional) Types the recipient's name
  T->>A: Taps "Submit"
  A->>B: Sends signature + finalises the report
  B-->>A: Report locked, timestamp recorded
  A-->>T: Shows the "Submitted" screen

  Note over T,B: What's next

  alt More items to handle
    T->>A: Taps "Scan another asset"
    A-->>T: Back to the tap screen
  else Done for now
    T->>A: Closes the app or signs out
  end
Read top to bottom. Arrows pointing right are actions or requests; arrows pointing back are responses. Boxes labelled alt / else show branches — the flow goes through one side or the other depending on the situation. Boxes labelled loop repeat for each photo the technician takes.
